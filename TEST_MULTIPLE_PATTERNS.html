Multiple Custom Patterns Feature Test

Feature Overview

This document describes the new Multiple Custom Patterns feature for the serial plot in the Remote Lab Interface.

What Changed:

  • Replaced single text boxes with dynamic pattern containers
  • Added "+ Add Pattern" buttons for both CH1 and CH2
  • Each pattern can be independently removed with the "×" button
  • Patterns are stored as arrays instead of single strings
  • Multiple patterns work seamlessly across all slides

How It Works

Step 1: Add a Pattern

Click the "+ Add CH1 Pattern" button to create a new pattern input field.

Step 2: Enter Pattern Format

Type your pattern using the {'{value}'} placeholder:

  • {'value}RPM'} - Extracts RPM data
  • temp: {'{value}'}°C - Extracts temperature
  • speed={'{value}'} - Custom format

Step 3: Press Enter to Confirm

The pattern is saved and appears in the value dropdown selector.

Step 4: Select & Plot

Choose the pattern from the dropdown to start plotting that data.

Step 5: Remove if Needed

Click the "×" button next to any pattern to remove it.

Example: Temperature Monitoring

Monitor temperature from multiple sensor formats:

Pattern 1 (CH1):

TEMP={'{value}'}

Matches: "TEMP=45.2" → Plots 45.2

Pattern 2 (CH1):

Temperature: {'{value}'}°C

Matches: "Temperature: 45.2°C" → Plots 45.2

Pattern 3 (CH1):

T:{'{value}'}

Matches: "T:45.2" → Plots 45.2

Result: All three patterns coexist, and you can select any to plot temperature data.

Cross-Slide Functionality

Technical Details

Data Structure:

serialPlotData.ch1.patterns = [
  { id: 'pattern_ch1_xxx', value: '{value}RPM' },
  { id: 'pattern_ch1_yyy', value: 'temp: {value}°C' }
]

Auto-Generated Pattern Names:

CH1_Pattern1 - First CH1 pattern

CH1_Pattern2 - Second CH1 pattern

CH2_Pattern1 - First CH2 pattern

CH2_Pattern2 - Second CH2 pattern

Key Functions

setupCustomFormatHandlers()

Initializes the pattern system and sets up event listeners.

addPatternInput(channel)

Dynamically adds a new pattern input field. Channel: 'ch1' or 'ch2'

renderPatterns(channel)

Renders all patterns from data structure with handlers.

parseValueFromFormat(data, format)

Extracts numeric value from serial data using pattern format.
